# 프롬프트 정책 규칙

## 목적

이 문서는 `turtle_agent`의 사용자 프롬프트, 시스템 프롬프트, 도구 독스트링, 장기기억 요약 프롬프트의 책임을 분리하기 위한 규칙을 정의한다.

프로젝트 목표는 로봇이 모든 문제 상황을 기본 규칙으로 미리 해결하는 것이 아니라, 관찰 가능한 정보와 장기기억에 근거해 행동을 개선하도록 만드는 것이다.

## 적용 범위

- 사용자 지시를 처리하는 프롬프트 계층
- `RobotSystemPrompts`에 전달되는 시스템 프롬프트
- LangChain tool 독스트링
- 장기기억에서 실행 정책을 요약해 주입하는 프롬프트
- 프롬프트 정책을 검증하는 테스트

## 1. 프롬프트 계층 책임

| 계층 | 책임 | 금지 사항 |
|---|---|---|
| 사용자 프롬프트 | 여러 터틀의 최상위 임무, 순서, 역할, 완료 조건을 지시한다. | 개별 로봇의 도구 호출 순서나 문제 해결 전략을 강제하지 않는다. |
| 시스템 프롬프트 | 모든 임무에 적용되는 보편 실행 규율을 제공한다. | 장애물 회피, 우회, 분절 이동처럼 특정 문제 해결 전략을 기본 정책으로 넣지 않는다. |
| 도구 독스트링 | 도구의 입력, 출력, 계산 능력, 한계를 설명한다. | 특정 상황에서 도구를 반드시 사용하라고 지시하지 않는다. |
| 장기기억 요약 프롬프트 | 선택된 장기기억에서 현재 임무에 적용 가능한 조건부 실행 근거를 요약한다. | 기억에 없는 일반 규칙이나 미래 문제 해결 전략을 만들지 않는다. |

## 2. 오픈소스 원본 시스템 프롬프트 보호

원 작성자가 작성한 시스템 프롬프트는 직접 수정하지 않는 것을 기본값으로 한다.

시스템 프롬프트가 프로젝트 정책을 심각하게 위반한다고 판단되는 경우에만 다음 절차를 따른다.

1. 수정 필요 문구와 정책 위반 근거를 분리해 기록한다.
2. `git diff`로 원본 대비 변경 범위를 확인한다.
3. 사용자에게 최소 수정안을 공유한다.
4. 사용자 명시 승인 후 최소 범위로 수정한다.

동일한 문제를 장기기억 요약, 프로젝트별 보조 정책, 도구 독스트링, 테스트로 해결할 수 있으면 그 경로를 우선한다.

## 3. 도구 독스트링 작성 규칙

mid-level 도구의 독스트링은 정책을 만들지 않고 고수준 정책과 저수준 실행 사이의 중간 표현과 판정 능력을 설명한다.

도구 독스트링에는 다음을 포함한다.

- 도구가 받는 입력
- 도구가 계산하거나 관찰하는 대상
- 반환값의 구조와 의미
- 도구가 하지 않는 일

도구 독스트링에는 다음을 포함하지 않는다.

- 특정 사용자 요청에서 반드시 호출하라는 문구
- 장기기억 없이도 특정 문제 해결 전략을 유도하는 문구
- 우회, 회피, 재계획 같은 다음 행동 강제 문구

## 4. 장기기억 요약 프롬프트 작성 규칙

장기기억 요약 프롬프트는 관찰된 근거만 현재 임무의 조건부 실행 근거로 바꾼다.

장기기억 요약에는 다음을 포함한다.

- 적용 가능한 목표 또는 작업 조건
- 실제 관찰된 성공, 실패, 충돌, 정체, 도구 결과
- 같은 조건이 재현될 때 참고할 수 있는 조건부 정책

장기기억 요약에는 다음을 포함하지 않는다.

- 관찰되지 않은 위험
- 모든 상황에 적용되는 보편 안전 규칙
- 근거 없는 도구 호출 지시
- 단순 도구 성공 여부나 호출 횟수 요약

## 5. 검증 기준

프롬프트 정책 변경은 다음 기준을 만족해야 한다.

- 장기기억이 없는 첫 실행에서 특정 문제 해결 전략이 프롬프트만으로 생성되지 않는다.
- 장기기억이 있는 경우에는 선택된 기억의 실제 내용만 실행 근거로 사용된다.
- 도구 독스트링은 호출 정책이 아니라 도구 능력 명세로 읽힌다.
- 원본 시스템 프롬프트 수정이 필요한 경우 사용자 승인과 `git diff` 확인 기록이 남는다.
